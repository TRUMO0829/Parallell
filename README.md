## Танилцуулга

Энэхүү төсөл нь 32-бит эрхийн санамсаргүй тоонуудыг **LSD Radix Sort**
алгоритмаар эрэмбэлдэг. 8 битийн алхамтай (RADIX = 256) 4 дамжлага хийнэ.
10,000 / 100,000 / 1,000,000 элементийн өгөгдөл дээр дараах хувилбаруудыг
харьцуулна:

1. **Sequential** — энгийн нэг утаст хувилбар.
2. **pthread** — POSIX утас ашигласан параллел хувилбар (C-д `std::thread`-ийн дүйцэл).
3. **OpenMP** — `#pragma omp` дириктив ашигласан хувилбар.
4. **CUDA** — *(хийгдээгүй)* GPU дээр ажиллах хувилбар.

> **Тэмдэглэл:** Одоогоор C хэлээр sequential, pthread, OpenMP гурван
> хувилбар хэрэгжүүлсэн. CUDA хувилбар нэмэхээр төлөвлөж байна.

## Хөрвүүлэх ба ажиллуулах

### Шаардлага
- GCC (≥ 7.0)
- POSIX threads (`-pthread`)
- OpenMP дэмжлэг (`-fopenmp`)

### Хөрвүүлэх

```bash
gcc -O2 -fopenmp -pthread -o radix_sort radix_sort_parallel.c
```

### Ажиллуулах

```bash
./radix_sort
```

Програм нь автоматаар 10K, 100K, 1M элементийн benchmark-ийг
дарааллуулан гүйцэтгэнэ.

## Хэрэгжүүлэлтүүд

### 1. Sequential — `radix_sort_sequential`
Сонгодог LSD Radix Sort:
1. **Count** — бункет бүрд харгалзах элементийн тоог тоолно.
2. **Prefix sum** — хуримтлагдсан нийлбэрийг бодно.
3. **Scatter** — арагшаа дамжуулан тогтвортой (stable) байдлаар хуулна.
4. **Copy back** — `tmp` буфераас гол массив руу буцаана.

### 2. pthread — `radix_sort_pthread`
- Массивыг `num_threads` хэсэгт хуваана.
- **Count** үе шатыг утас тус бүр локал `local_count[256]` руу
  зэрэгцээ гүйцэтгэнэ → синхрончлолын зардалгүй.
- Утаснуудыг `pthread_join` хийсний дараа локал тоологчдыг нэгтгэнэ.
- **Scatter** үе шат нь тогтвортой байдлыг хадгалахын тулд цуваа хийгдэнэ.

### 3. OpenMP — `radix_sort_openmp`
- `#pragma omp parallel` бүсэд утас бүр өөрийн `local_counts[tid]`-ыг
  бөглөнө (`schedule(static)`).
- `#pragma omp single` хэсэгт буферийг нэг удаа хуваарилна.
- Нэгтгэх ба scatter үе шатууд цуваа.

## Тохиргоо

`radix_sort_parallel.c` дотор тохируулж болох тогтмолууд:

| Тогтмол | Утга | Тайлбар |
|---|---|---|
| `BITS` | 32 | Тоонуудын бит өргөн |
| `RADIX` | 256 | Нэг алхамд боловсруулагдах бункетийн тоо (8 бит) |
| `PASSES` | 4 | Дамжлагын тоо (BITS / 8) |
| `MAX_THREADS` | 8 | pthread-ын дээд хязгаар |

Benchmark нь pthread болон OpenMP хувилбарыг **2, 4, 8** утсаар тус тус туршина.

## Бие даалтын даалгавар

### Ажлын зорилго

Бие даалтын ажлын хүрээнд эрэмбэлэлтийн алгоритмыг нийт **4 өөр хувилбараар**
хэрэгжүүлж, тэдгээрийн гүйцэтгэлийг харьцуулан шинжлэх замаар параллел
тооцооллын үндсэн технологиудыг эзэмшихэд оршино.

### Ажлын даалгавар

Баг нь бие даалтыг гүйцэтгэхдээ алгоритмыг дараах **4 өөр хувилбараар**
програмчилж, **10K, 100K, 1M** элементүүд дээр үр дүнг харьцуулна:

1. Энгийн дараалсан хувилбар (**sequential**).
2. CPU-ийн олон цөмийг ашигласан стандарт сангийн хувилбар (**`std::thread`**).
3. Compiler дириктив ашиглан оновчилсон CPU-ийн параллел хувилбар (**OpenMP**).
4. GPU-ийн цөмийг ашигласан архитектурт суурилсан хувилбар (**CUDA**).

### Тайлангийн шаардлага

Тайланг эрдэм шинжилгээний өгүүлэл хэлбэртэй буюу дараах бүтцийг агуулсан
байдлаар бэлтгэнэ:

1. **Хураангуй** — Ямар ажил хийгээд, ямар тодорхой үр дүнд хүрсэнээ
   товчхон бичнэ.
2. **Онолын ойлголтууд** — Шинэ алгоритмыг боловсруулахад онолын
   материалуудыг оруулна. Үндсэн сурах бичиг болон нэмэлт материалуудыг
   ашиглаж болно.
3. **Алгоритмын зохиомж** — Шинэ алгоритмуудын ажиллагааны зарчмыг зургаар
   үзүүлж, зангилаа, санах ой, кэш, регистрийн түвшинд ойлгомжтой
   тайлбарласан байна. Гол функцуудын кодыг псевдо кодоор оруулж тайлбарлана.
4. **Үр дүн, харьцуулсан шинжилгээ** — *SpeedUp*, *Computation time*,
   *Total operations*, *Data transfer time*, *Amount of data to be transferred*,
   *Execution time*, *Achievable performance* зэрэг үнэлгээнүүдийг шинэ,
   хуучин алгоритм тус бүрт гаргаж харьцуулна.
5. **Дүгнэлт** — Үр дүн, харьцуулсан шинжилгээнд үндэслэн ямар амжилтанд
   хүрсэнийг бичнэ.
6. **Ашигласан материалын жагсаалт** — Онолын судалгаа болон алгоритмын
   зохиомжийг хийхэд хэрэглэсэн эх сурвалжийн жагсаалт (IEEE/APA стандарт).
